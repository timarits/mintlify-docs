---

Creates a new project and associated credit wallet for the authenticated user.

<ParamField body="name" type="string" required>
  Project name for identification
</ParamField>

<RequestExample>
```bash cURL
curl -X POST https://your-project.supabase.co/functions/v1/api/v1/projects \
  -H "Authorization: Bearer your-anon-key" \
  -H "Content-Type: application/json" \
  -H "Idempotency-Key: create-project-123" \
  -d '{
    "name": "My Integration Project"
  }'
```

```typescript SDK
import { createApiClient } from '@partner-funded-credits/sdk';

const client = createApiClient({
  baseUrl: 'https://your-project.supabase.co/functions/v1/api',
  apiKey: 'your-anon-key'
});

const project = await client.createProject({
  name: 'My Integration Project'
});
```

```javascript JavaScript
const response = await fetch('https://your-project.supabase.co/functions/v1/api/v1/projects', {
  method: 'POST',
  headers: {
    'Authorization': 'Bearer your-anon-key',
    'Content-Type': 'application/json',
    'Idempotency-Key': 'create-project-123'
  },
  body: JSON.stringify({
    name: 'My Integration Project'
  })
});

const project = await response.json();
```
</RequestExample>

## Response Fields

<ResponseField name="project_id" type="string">
  Unique identifier for the created project
</ResponseField>

<ResponseField name="wallet_id" type="string">
  Unique identifier for the associated credit wallet
</ResponseField>

<ResponseField name="balance" type="number">
  Initial credit balance (always 0 for new projects)
</ResponseField>

## Error Responses

<ResponseField name="401 Unauthorized" type="object">

  ```json

    "error": "unauthorized",
    "details": "Missing or invalid Authorization header"
  ```

<ResponseField name="400 Bad Request" type="object">

</ResponseField>

## Notes

<Info>
Projects are automatically associated with the authenticated user via Row-Level Security (RLS). Users can only access their own projects.
</Info>

<Warning>
Project names should be unique per user for better organization, though this is not enforced at the database level.
</Warning>

## Idempotency

This endpoint supports idempotency. Include an `Idempotency-Key` header to safely retry requests:

```bash
curl -H "Idempotency-Key: create-project-$(date +%s)" \
  # ... rest of request
```

Using the same idempotency key will return the original project without creating a duplicate.
